What is the OSI Model?

A conceptual framework organizing network communication into seven distinct layers

Each layer handles a specific aspect: physical transmission, routing, reliable delivery, application protocols

Layers provide services to the layer above and consume services from below

Separation allows protocols to change without affecting other layers

Layers 1-2: Physical & Data Link

Layer 1 (Physical): Raw bit transmissionâ€”electrical signals, cables, fiber optics, wireless radios

No understanding of addresses or packets; simply transmits/receives bit streams

Layer 2 (Data Link): Organizes bits into frames; uses MAC addresses for local delivery

Switches operate at Layer 2, forwarding frames based on MAC addresses

Layer 3: Network Layer

Enables routing across multiple networks using IP addresses

Routers operate at Layer 3, consulting routing tables to forward packets

Handles packet fragmentation when data exceeds link capacity

ICMP provides diagnostics (ping, traceroute)

Layer 4: Transport Layer

Manages end-to-end communication; uses port numbers to identify applications

TCP: Connection-oriented, guaranteed delivery, sequence tracking, retransmission

UDP: Connectionless, no delivery guarantee, lower latency, less overhead

Choose TCP when data integrity matters; UDP when speed matters more than occasional loss

Layers 5-7: Session, Presentation, Application

Layer 5 (Session): Manages connection establishment, maintenance, and termination

Layer 6 (Presentation): Data format translation, encryption/decryption, compression

Layer 7 (Application): Network services for user applicationsâ€”HTTP, HTTPS, DNS, SSH, SMTP

Many applications integrate session/presentation directly into application protocols

Load Balancers: Layer 4 vs Layer 7

Layer 4 Load Balancer: Routes based on IP addresses and ports; fast, minimal inspection

Layer 7 Load Balancer: Parses HTTPâ€”routes by URL path, headers, cookies; enables sophisticated rules

L7 can route /api/* to API servers, /static/* to CDN; terminate SSL; inspect for threats

Many deployments use both: L4 distributes to L7 balancers, which route to backends
