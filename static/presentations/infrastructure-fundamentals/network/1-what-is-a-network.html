Infrastructure Fundamentals

The Network Concept

A network connects computing devices (nodes) to enable data exchange and resource sharing

Data travels in discrete units called packets containing payload and routing information

LAN (Local Area Network): devices within a building/campus; WAN (Wide Area Network): spans cities/continents

The internet is a global network of interconnected networks using standardized protocols

Key Network Components

Routers: Direct data between networks using IP addresses; determine optimal packet paths

Switches: Manage data within local networks based on MAC addresses

Firewalls: Monitor and control traffic using security rules

Wireless Access Points: Bridge wireless devices to wired infrastructure

Load Balancers & Reverse Proxies: Distribute traffic and enhance performance

Network Topologies

Bus: Shared communication line; cost-effective but limited fault tolerance

Star: Devices connect to central hub; isolates faults effectively (dominates modern LANs)

Ring: Circular connection; predictable performance but vulnerable to breaks

Mesh: Multiple redundant paths; maximizes reliability but requires more infrastructure

Network Abstraction Layers

Physical Layer: Tangible infrastructureâ€”cables, NICs, switches, routers

Virtual Networks: VLANs create logical segments on shared physical infrastructure

Cloud Networks: Azure VNets/AWS VPCs provide isolated environments via software-defined networking

Serverless Networking: Platform handles connectivity, load balancing, and scaling automatically

Core Network Functions

Connectivity & Routing: Establishes paths; routers forward packets hop-by-hop toward destinations

Switching & Segmentation: Switches forward within LANs; VLANs isolate traffic for security

Protocol-Governed Communication: TCP/IP suite provides foundation; HTTP, DNS, SSH build on top

Traffic Management: Load balancers distribute requests; reverse proxies handle SSL termination and caching

Network Security Mechanisms

Firewalls: Filter traffic based on IP addresses, ports, and protocols

Network Security Groups: Cloud-native filtering at interface/subnet level

IDS/IPS: Monitor for suspicious patterns; alert or actively block threats

Encryption (TLS/VPNs): Protect data in transit; create secure tunnels through untrusted networks
