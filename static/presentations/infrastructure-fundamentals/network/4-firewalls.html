What is a Firewall?

A firewall acts as a barrier between trusted internal networks and untrusted external ones

Examines each packet crossing the network boundary and decides to permit or block

Decisions follow rules defined by administrators based on packet attributes

If no rule matches, a default policy determines the outcome (typically "deny all")

The Five-Tuple

Firewall rules commonly match on five packet attributes (the five-tuple):

Source IP address: The originating system's network address

Destination IP address: The target system's network address

Source/Destination ports: Identify the service (e.g., 443 for HTTPS)

Protocol: Transport protocolâ€”typically TCP or UDP

Ports and Services

Network services listen on specific portsâ€”numeric identifiers directing traffic to applications

Port 22: SSH (secure remote access)

Port 80/443: HTTP/HTTPS (web traffic)

Port 3306: MySQL; Port 5432: PostgreSQL

Port-based filtering grants or denies access to entire services

Firewall Architectures

Packet-Filtering: Examines packets individually; fast but no connection awareness

Stateful Inspection: Tracks connection state; validates traffic belongs to established sessions

Next-Generation (NGFW): Application-layer awareness, deep packet inspection, threat detection

Most modern firewalls operate statefully by default for meaningful security
