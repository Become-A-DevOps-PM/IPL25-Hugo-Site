Infrastructure Fundamentals

The Address Space Problem

IPv4 provides approximately 4.3 billion unique addressesâ€”insufficient for modern device proliferation

Rather than require globally unique addresses for every device, standards define private ranges

Private addresses communicate within local networks but cannot route across the public internet

This preserves public addresses for systems needing global reachability

Private Networks

Use non-routable IP addresses for internal communication (RFC 1918)

10.0.0.0/8: ~16 million addresses for large organizations

172.16.0.0/12: ~1 million addresses for medium networks

192.168.0.0/16: 65,536 addresses common in home/small office

These ranges can be reused across organizations without conflict

How Private Networks Operate

Devices receive private IPs (e.g., 192.168.1.10, 192.168.1.20) and communicate directly

NAT (Network Address Translation) rewrites source addresses when traffic leaves the network

Multiple devices share a single public IPâ€”external systems see only the router's address

Router maintains translation table mapping internal devices to connections

Public Networks

Use globally unique, routable IP addresses assigned by ISPs/cloud providers

Static public IPs: Remain constant; ideal for servers with DNS records

Dynamic public IPs: May change on reconnection; common for residential connections

Any system with internet connectivity can potentially reach public addresses

Connectivity and Exposure

Public IPs are globally reachableâ€”enables services but requires robust security

Firewalls filter incoming connections, allowing only specific ports/protocols

Network Security Groups define rules before traffic reaches resources

DMZ (Demilitarized Zone): Public-facing systems isolated from internal network

Choosing Between Private and Public

Public IP: Web servers, jump boxes, API gatewaysâ€”must accept external connections

Private IP: Databases, application servers, internal file serversâ€”no direct internet access needed

Common pattern: Load balancer with public IP distributes to private backend servers

Only expose what must be public; everything else operates on private networks
