STORAGE

Infrastructure Fundamentals

What is Storage?

Storage refers to saving and accessing data for future use

Critical for applications, systems, and data-driven operations

Distinct from databases: storage provides the underlying infrastructure, databases add structure and query capabilities

Foundation of the persistence layer in any system architecture

Block Storage

Divides data into fixed-sized blocks

Each block has a unique address, accessed independently

Used for databases and applications requiring direct disk access

Examples: AWS EBS, Azure Disk Storage, local SSDs/HDDs

Protocols: iSCSI, Fibre Channel, NVMe

File Storage

Organizes data in a hierarchical structure with files and folders

Ideal for shared access across multiple systems

Network-attached storage (NAS) provides centralized file services

Protocols: NFS (Unix/Linux), SMB/CIFS (Windows)

Examples: Azure Files, Amazon EFS, traditional NAS appliances

Object Storage

Stores data as objects with metadata and unique IDs

No hierarchical folder structure - flat namespace with buckets/containers

Suitable for unstructured data at massive scale

Accessed via HTTP/S APIs (REST)

Examples: AWS S3, Azure Blob Storage, Google Cloud Storage

Storage in Cloud Environments

Cloud Storage Services: Provide scalable and managed solutions

Examples: Google Cloud Storage, Amazon S3, Azure Blob Storage

Hybrid Storage: Combines on-premises and cloud storage for flexibility

Pay-per-use pricing with multiple tiers (hot, cool, archive)

Key Storage Concepts

Capacity: The total amount of data storage available (GB, TB, PB)

Throughput: Amount of data transferred per second (MB/s, GB/s)

Latency: The time it takes to access stored data (ms, Î¼s)

Redundancy: Ensures data availability during hardware failures (RAID, replication)

Challenges and Benefits

Scalability: Meeting growing data demands without downtime

Security: Protecting sensitive information with encryption and access controls

Performance: Balancing speed and resource usage for workload requirements

Benefits: Disaster recovery, accessibility across devices, support for analytics

Practical Applications

Data lakes: Big data analysis and machine learning workloads

Media storage: Video streaming platforms, image repositories

Backup and archival: Long-term data retention and compliance

Application data: VM disks, container volumes, database storage
