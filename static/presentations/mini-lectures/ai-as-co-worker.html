A Mental Model for AI Collaboration

DevOps Project Management • 2025

The Key Shift

Stop asking: "Can AI do this task?"

Start asking: "How should I brief this team member?"

AI agents process context, apply specialized knowledge, and produce substantive work—more like a specialist consultant than software.

Tools have features to learn. Colleagues have strengths to leverage and working relationships to develop.

Three Modes of Interaction

Ask Mode: Quick lookups, syntax verification, reference details. Single exchange, minimal context needed. Like asking a colleague for quick confirmation.

Investigate Mode: Error diagnosis, configuration review, architecture evaluation. Builds shared understanding. Output informs decisions rather than executing them.

Plan-Execute Mode: Producing code, configurations, documentation. Extended interaction with iteration. Human maintains decision authority while AI handles technical execution.

Most people start with Ask, then gradually discover they can delegate more sophisticated work.

Context Determines Quality

Intent Context: What you aim to accomplish and why. Enables appropriate design choices rather than generic commands.

Technology Stack Context: Exact technologies and versions. PostgreSQL 14 differs from 12; Azure CLI differs from AWS CLI.

Tool & Environment Context: Existing workflows, constraints (no root access, compliance requirements). Shapes output form and prevents unusable proposals.

The difference between generic output and genuinely useful output is context.

Calibrating Trust

Key difference from human colleagues: AI does not accumulate wisdom. Each session starts without memory of previous interactions.

AI Strengths: Pattern-based tasks, structured formats, standard patterns, documentation synthesis, iterative refinement, execution and verification.

AI Limitations: Training data cutoff, no cross-session memory, limited detection of subtle logical errors, cannot apply undocumented organizational judgment.

Scale verification to risk: Critical scripts warrant thorough testing. Documentation drafts require accuracy checks but carry less risk.

Key Takeaways

Shift your mental model. Brief AI like a team member, not a search engine.

Match mode to task. Ask for lookups, Investigate for analysis, Plan-Execute for deliverables.

Context determines quality. Provide intent, stack, and tool context.

Treat collaboration as a skill. Effectiveness improves with practice—structuring requests, evaluating outputs, iterating efficiently.
