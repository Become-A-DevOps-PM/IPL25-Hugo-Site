This week focuses on building a complete Flask web application using three-tier architecture. You progressively build a full-featured newsletter subscription application with proper separation of concerns.

## Practice

### Three-Tier Architecture

Build **News Flash**, a newsletter subscription application through a series of progressive exercises:

- [Three-Tier Architecture Overview](/exercises/application-development/1-three-tier-architecture/) - Introduction to the exercise series and architecture concepts
- [Project Structure and Application Factory](/exercises/application-development/1-three-tier-architecture/2-project-structure-and-application-factory/) - Organize a Flask application using three-tier architecture and the application factory pattern
- [Template Inheritance and Your First Route](/exercises/application-development/1-three-tier-architecture/3-template-inheritance-and-first-route/) - Build a landing page using Jinja2 template inheritance and Flask Blueprints
- [Hero Section and Interactive Modal](/exercises/application-development/1-three-tier-architecture/4-hero-section-and-interactive-modal/) - Build a visually engaging landing page with a styled hero section and an accessible modal dialog
- [Subscription Form and Thank You Page](/exercises/application-development/1-three-tier-architecture/5-subscription-form-and-thank-you-page/) - Build a subscription form and handle form submissions with a confirmation page
- [Business Layer Validation Service](/exercises/application-development/1-three-tier-architecture/6-business-layer-validation-service/) - Build a validation service in the business layer to validate and normalize subscription data
- [Database Setup and Subscriber Model](/exercises/application-development/1-three-tier-architecture/7-database-setup-and-subscriber-model/) - Build the data layer foundation by configuring SQLAlchemy and creating database migrations
- [Repository Pattern and Full Integration](/exercises/application-development/1-three-tier-architecture/8-repository-pattern-and-full-integration/) - Build a repository class and integrate all three layers for complete data persistence
